# SynthCore 框架结构

## 项目结构概览

SynthCore 采用模块化架构设计，将不同功能组件分离为独立的模块，便于维护和扩展。以下是项目的主要目录结构：

```
├── src/              # 核心源代码
│   ├── agents/       # 代理相关代码
│   ├── browser/      # 浏览器控制
│   ├── channels/     # 消息渠道集成
│   ├── cli/          # 命令行界面
│   ├── config/       # 配置管理
│   ├── gateway/      # 网关服务
│   ├── media/        # 媒体处理
│   ├── plugins/      # 插件系统
│   └── utils/        # 工具函数
├── extensions/       # 扩展插件
├── docs/             # 文档
├── apps/             # 应用程序
│   ├── android/      # Android 节点
│   ├── ios/          # iOS 节点
│   └── macos/        # macOS 应用
├── skills/           # 技能系统
├── scripts/          # 脚本工具
├── test/             # 测试代码
└── ui/               # Web UI
```

## 核心模块

### 1. Gateway

**位置**：`src/gateway/`

**功能**：作为整个系统的控制平面，管理会话、通道、工具和事件。

**主要文件**：
- `server.ts`：Gateway 服务器实现
- `client.ts`：Gateway 客户端实现
- `events.ts`：事件处理
- `auth.ts`：认证管理

**核心功能**：
- WebSocket 服务器，处理客户端连接
- 会话管理和状态维护
- 通道集成和消息路由
- 工具执行和管理
- 事件分发和处理

### 2. Agents

**位置**：`src/agents/`

**功能**：智能代理，处理用户请求并执行相应的操作。

**主要文件**：
- `context.ts`：上下文管理
- `defaults.ts`：默认配置
- `skills.ts`：技能管理

**核心功能**：
- 处理用户输入和生成回复
- 管理对话上下文
- 执行技能和工具
- 处理多轮对话

### 3. Channels

**位置**：`src/channels/`

**功能**：多渠道集成，支持各种消息平台。

**主要文件**：
- `dock.ts`：通道管理
- `typing.ts`：输入状态管理

**支持的渠道**：
- WhatsApp

- Slack
- Discord

- IRC

- Matrix

- LINE

- Twitch

- Zalo Personal
- WebChat

### 4. Plugins

**位置**：`src/plugins/` 和 `extensions/`

**功能**：插件扩展系统，允许添加新功能和集成。

**主要文件**：
- `loader.ts`：插件加载器
- `runtime.ts`：插件运行时

**插件类型**：
- 认证插件：如 Auth0、AWS、Azure 等
- 存储插件：如 Google Drive、Dropbox、OneDrive 等
- 集成插件：如 GitHub、GitLab、Jira 等
- 技能插件：如天气、新闻、财务等
- 工具插件：如加密、备份、监控等

### 5. Tools

**位置**：`src/` 下的各个工具模块

**功能**：内置工具，提供各种功能。

**主要工具**：
- **Browser**：浏览器控制，自动化网页操作
- **Canvas**：可视化工作空间
- **Nodes**：设备控制和操作
- **Cron**：定时任务管理
- **Media**：媒体处理
- **Memory**：内存管理

### 6. CLI

**位置**：`src/cli/`

**功能**：命令行界面，提供用户交互。

**主要文件**：
- `program.ts`：命令行程序入口
- `route.ts`：命令路由
- `prompt.ts`：命令提示

**主要命令**：
- `openclaw gateway`：管理网关
- `openclaw agent`：与代理交互
- `openclaw message`：发送消息
- `openclaw plugins`：管理插件
- `openclaw skills`：管理技能
- `openclaw config`：管理配置

### 7. Config

**位置**：`src/config/`

**功能**：配置管理，处理系统配置。

**主要文件**：
- `config.ts`：配置管理
- `schema.ts`：配置 schema
- `defaults.ts`：默认配置

**配置存储**：
- `~/.openclaw/openclaw.json`：主配置文件
- 环境变量：覆盖配置

### 8. Media

**位置**：`src/media/`

**功能**：媒体处理，处理图片、音频和视频。

**主要文件**：
- `host.ts`：媒体主机
- `parse.ts`：媒体解析
- `store.ts`：媒体存储

**核心功能**：
- 媒体上传和下载
- 媒体转码和处理
- 媒体分析

### 9. Apps

**位置**：`apps/`

**功能**：客户端应用程序。

**主要应用**：
- **macOS**：菜单 bar 应用，提供控制界面
- **iOS**：iOS 节点，提供移动功能
- **Android**：Android 节点，提供移动功能

### 10. Skills

**位置**：`skills/` 和 `extensions/*/skills/`

**功能**：技能系统，扩展代理能力。

**主要技能**：
- 天气技能
- 新闻技能
- 财务技能
- 任务技能
- 旅行技能
- 食物技能

## 数据流

### 1. 用户请求流程

1. 用户通过消息渠道发送请求
2. 渠道接收消息并转发到 Gateway
3. Gateway 路由消息到相应的代理
4. 代理处理请求并生成回复
5. Gateway 将回复路由回相应的渠道
6. 渠道将回复发送给用户

### 2. 插件加载流程

1. Gateway 启动时加载插件
2. 插件注册到系统
3. 代理可以调用插件提供的功能
4. 插件可以响应系统事件

### 3. 工具执行流程

1. 代理决定执行某个工具
2. Gateway 调用相应的工具
3. 工具执行并返回结果
4. 代理处理结果并生成回复

## 扩展机制

### 1. 插件扩展

- **插件结构**：每个插件包含 `openclaw.plugin.json` 配置文件和实现代码
- **插件加载**：Gateway 启动时自动加载 `extensions/` 目录下的插件
- **插件 API**：插件可以实现各种接口，如认证、存储、集成等

### 2. 技能扩展

- **技能结构**：每个技能包含 `SKILL.md` 描述文件和实现代码
- **技能加载**：代理启动时加载技能
- **技能调用**：代理可以根据用户请求调用相应的技能

### 3. 渠道扩展

- **渠道实现**：实现 `Channel` 接口
- **渠道注册**：在 Gateway 中注册渠道
- **渠道配置**：通过配置文件配置渠道参数

## 部署架构

### 1. 本地部署

- **Gateway**：运行在本地机器上
- **客户端**：通过 WebSocket 连接到 Gateway
- **存储**：本地文件系统

### 2. 远程部署

- **Gateway**：运行在远程服务器上
- **客户端**：通过 Tailscale 或 SSH 隧道连接到 Gateway
- **存储**：远程存储或云存储

### 3. 容器部署

- **Docker**：使用 Docker 容器运行 Gateway
- **Docker Compose**：使用 Docker Compose 管理多个服务

## 总结

SynthCore 采用模块化、可扩展的架构设计，通过 Gateway 作为控制平面，集成多个消息渠道，提供智能代理服务。系统支持插件扩展、技能系统和多平台客户端，为用户提供全方位的智能助手体验。
